# ML-HR

뉴스 요약 모델 인사고과 및 자동 배포 시스템

## 1. 프로젝트 개요

### 배경
뉴스 데이터는 신조어 발생과 이슈 전환이 매우 빠릅니다. 이 때문에 학습 시점과 운영 시점의 데이터 분포가 달라지는 `Data Drift`가 발생할 수 있고, 기존 요약 모델의 성능이 빠르게 저하될 수 있습니다.

### 목표
단일 모델에 의존하지 않고, 여러 모델 후보를 체계적으로 평가해 최적의 모델을 선발하고 운영하는 MLOps 파이프라인을 구축합니다.

## 2. 핵심 아이디어

- 5개 모델 후보를 동일한 기준으로 평가
- 성능, 응답 속도, 비용을 종합해 우수 모델 선발
- 선발된 모델을 API로 배포
- 운영 중 데이터 드리프트와 서비스 상태를 지속적으로 모니터링

## 3. 기술 스택

### DataOps
- `Git`
  - 소스 코드 버전 관리
- `DVC`
  - 대용량 뉴스 데이터셋 및 모델 가중치 버전 관리

### ModelOps
- `MLflow`
  - 모델별 실험 결과 기록
  - ROUGE Score, Latency, 비용 기반 비교
  - 우수 모델을 Model Registry에 등록
- `SageMaker Studio Lab`
  - 모델 학습 및 실험 수행 환경

### DevOps
- `FastAPI`
  - 모델 서빙용 REST API 구현
- `Docker`
  - 실행 환경 표준화 및 배포 편의성 확보
- `Render`
  - 컨테이너 기반 자동 배포 및 실시간 서비스 운영

### Monitoring
- `Evidently AI`
  - 뉴스 트렌드 변화에 따른 데이터 드리프트 시각화
- `Prometheus`
  - 서버 및 추론 지표 수집
- `Grafana`
  - 서비스 상태와 성능 대시보드 구성

## 4. 데이터셋 및 분석 계획

### 사용 데이터
- `AI-Hub 문서요약 텍스트`
  - 학습용 데이터
- `네이버 뉴스 API`
  - 실시간 테스트용 데이터

### EDA 계획
- 뉴스 본문 길이와 요약문 길이 분포 분석
- 모델별 입력 제한(Token Limit) 설정
- 카테고리별 데이터 불균형 여부 확인

### 전처리 계획
- 특수문자 정리
- 불용어 처리
- 모델별 전용 토크나이저 적용
- 재현 가능한 전처리 파이프라인 구축

## 5. 모델 후보군

### 1) KoBART
한국어 뉴스 요약에 적합한 모델로, 경량성과 효율성을 중심으로 평가합니다.

### 2) KoT5
자연스러운 문장 생성 능력을 강점으로 하는 모델입니다.

### 3) GPT-3.5 API
유료 대형 모델을 기준점으로 두어 성능 벤치마크에 활용합니다.

### 4) KLUE-RoBERTa
핵심 문장 추출 성능 검증용 기준 모델로 활용합니다.

### 5) Mistral-7B (Lite)
오픈소스 LLM의 뉴스 처리 및 요약 가능성을 검증합니다.

## 6. 기대 결과

- 다양한 모델을 동일 조건에서 비교 가능한 평가 체계 확보
- 성능 저하에 대응 가능한 모델 운영 프로세스 구축
- 모델 선발부터 배포, 모니터링까지 연결된 MLOps 실습 경험 확보
